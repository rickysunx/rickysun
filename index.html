The Service Monitoring System was created for monitoring the game service platform, checking whether the services running well all the time (7*24). The monitor system alerts the developers or operators by sending short messages and emails while it found anything unstable or exceptions. The monitor system checked the game service in two ways, by statistics and analysis of the logs sent by the game service platform system, or by checking the statuses of the http requests to the services provided by the game service platform system. The monitor system can be divided into two modules: the real-time analyzing and alerting module which monitoring the indicators every second and sending the alerts while exceptions occurred; the management module which is a user interface based on web for the developers or manager to create or edit the indicators and the reports of the exception list can be also found here. There are nearly hundreds of games running depending on the services provided by the platform system; therefore the amount of the logs sent from the platform system to the monitor system is pretty large, about 300 logs per second in general situation and 3000 logs per second in rush hours. Due to these situations, the monitor system should be designed for high performance. With the help of thread pooling and concurrent computing technology, I divided the analyzing task into pieces, and then submitted them to the multiple processors on the server. I designed a kind of memory pool to buffer the data waiting for statistics later, which can decrease memory fragments. The system was built based on Spring MVC. The front-end of the system was based on ExtJS, and I used JFreeChart to generate charts of the reports.
The Accounting Analysis and Reporting System extracts data from Yonyou Accounting System, analyses the data by user-defined rules and then presents the reports which contain bar chart, pie chart, line chart and spread sheets. The analysis system consists of indicator definition, data extraction functions, indicator formula definition, formula executor, data processor, reports query, and chart query modules. The system consists of about 105,553 lines of Java code and it was based on Yonyou Application Platform. The customers include Dalian Bank and Fuzhou Bank.
The Consumable Management System was built for bank customers managing their consumable items such as laptops, office supplies. It provided a more easy and convenient way than managing those items in Assert Management System. The system could also generate accounting data automatically to Yonyou Accounting System reducing a lot of extra work for accountants. The system consists of user management, role management, department management, accounting subjects setting, indicator setting, budget reports designer, budget reports input, reports gathering, and budget usage reports modules. The customers include Ningbo bank, Zhejiang rural bank, Shijiazhuang bank and Huludao bank.
The Budget Management System was used for making, managing and controlling the budget plans. It was designed for accountants in the bank to making budget plan, and could be integrated with the financial accounting system to control the voucher creation. The users could also review the reports of the budget usage and balance in the system. The system consisted of user management, role management, department management, accounting subjects setting, indicator setting, budget reports designer, budget reports input, reports gathering, and budget usage reports modules. The customers include Ningbo bank, Zhejiang rural bank, Shijiazhuang bank and Huludao bank.
The Project and Document Management System (PDMS) was built for the project managers to manage the projects and share documents. The PDMS was able to extract raw texts from the uploaded Word, Excel and PowerPoint documents via Apache POI library, and it also provided search services via Apache Lucene. The system converted the documents into Shockwave Flash format by invoking Macromedia FlashPaper automatically so the users could view the documents in the browsers without downloading them into local file system.
The Budget Management System (BMS) for life insurance industry was built from scratch. BMS integrated with the existed Sinosoft Accounting System and the users were able to configure criteria for monitoring and checking accounting data. BMS would trigger a warning or prevent transactions from being processed when it ran out of budgets.
